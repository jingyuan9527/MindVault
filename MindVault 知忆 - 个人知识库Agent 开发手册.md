# 一、项目概述

**MindVault 知忆** 是一个能主动帮助用户消化、整理、关联知识的智能助手。区别于传统笔记软件，本系统利用AI Agent能力实现知识的自动分类、智能检索、关联发现、间隔复习等高级功能。

## 🎯 核心价值

- 任何形式输入一键沉淀（文字/链接/PDF/图片）

- 自然语言检索，不用记文件夹位置

- 自动发现知识关联，构建知识网络

- 艾宾浩斯遗忘曲线智能提醒复习

- 基于个人知识库的智能写作辅助

- 完整导入导出，数据完全自主可控

- 多模型供应商支持，灵活切换主备模型

## 💡 设计原则

- 单Agent优先，避免过度设计

- 渐进式迭代，每个版本都完整可用

- 纯Web端交互，无命令行依赖

- Java原生技术栈，JDK 21 新特性

- Docker一键部署，开箱即用

- 数据可导入导出，不锁定用户

- 多模型支持，避免供应商锁定

# 十、Agent 知识处理时机与调度设计

**核心问题：**Agent什么时候对笔记进行整理、摘要、打标签、关联发现？

## 10\.1 三种处理模式对比

|处理模式|触发时机|优点|缺点|适用场景|
|---|---|---|---|---|
|**实时处理**|用户添加知识**立即**处理|即时可用，用户体验好|阻塞用户操作，大文件卡顿|短文本、URL|
|**异步后台处理**|添加后放入队列，**后台**处理|不阻塞用户，体验流畅|处理有延迟，需要状态显示|PDF、长文档|
|**定时批量处理**|**固定时间**（如每日凌晨2点）|集中处理，削峰填谷|当天添加的知识当天不可用|知识关联、复习推荐|

## 10\.2 推荐设计：混合模式

### 第一级：入库即处理（实时）

**触发时机：**用户添加知识的同时

- ✅ 生成向量嵌入（必须实时，否则无法检索）

- ✅ 短文本（\< 500字）直接生成摘要和标签

- ✅ URL网页标题抓取

### 第二级：异步队列处理（准实时）

**触发时机：**入库后立即放入后台任务队列

- ⏳ 长文本/大文档的要点提取

- ⏳ PDF全文解析与分片

- ⏳ 智能标签生成（需要LLM深度理解）

- ⏳ 前端显示"处理中\.\.\."状态，完成后刷新

### 第三级：定时批量处理（每日）

**触发时机：每日凌晨 02:00 \- 04:00**（用户低峰期）

|任务|执行时间|说明|
|---|---|---|
|**知识关联发现**|02:00|计算今日新增知识与历史知识的相似度，建立关联|
|**复习计划生成**|02:15|根据艾宾浩斯曲线，生成今日待复习列表|
|**每日知识复盘**|02:30|生成今日新增知识的摘要报告|
|**数据自动备份**|03:00|全量JSON备份，清理7天前旧备份|
|**Token用量统计**|03:30|统计昨日用量，生成成本报告|
|**向量一致性检查**|03:45|检查是否有缺失向量的知识，自动补全|

# 十一、检索策略与降级机制设计

**核心原则：**支持混合检索，无向量模型时优雅降级，绝对不能报错

## 11\.1 三级检索策略（自动判断）

|策略|触发条件|检索方式|
|---|---|---|
|**智能混合检索**|已配置嵌入模型|关键字检索 \+ 向量相似度检索 \+ RRF重排序|
|**纯向量检索**|已配置嵌入模型，用户指定|仅余弦相似度匹配|
|**纯关键字检索**|未配置嵌入模型 OR 向量检索失败|PostgreSQL全文检索 \+ LIKE模糊匹配|

## 11\.2 降级机制（核心设计）

**铁律：**即使用户完全不配置任何LLM模型，系统也必须能正常运行！

### 降级判断逻辑

```
// 1. 检查是否配置了嵌入模型
boolean hasEmbeddingModel = modelConfigService.hasValidEmbeddingModel();

if (hasEmbeddingModel) {
    try {
        // 2. 尝试混合检索
        return hybridSearch(query, topN);
    } catch (Exception e) {
        // 3. 向量检索失败，自动降级到关键字检索
        log.warn("向量检索失败，自动降级到关键字检索: {}", e.getMessage());
        return keywordSearch(query, topN);
    }
} else {
    // 4. 未配置向量模型，直接关键字检索
    log.info("未配置嵌入模型，使用关键字检索");
    return keywordSearch(query, topN);
}
```

### 三种模型独立配置

|模型类型|用途|是否必填|降级策略|
|---|---|---|---|
|**对话模型**|Agent聊天、回答问题|⭐ 推荐|降级为纯检索模式，不生成回答|
|**嵌入模型**|生成向量、语义检索|可选|✅ 降级为纯关键字检索|
|**摘要模型**|生成摘要、标签、关联发现|可选|降级为无摘要、无智能标签|

**最小编可用配置：**什么模型都不配置 → 系统退化为一个带全文检索的笔记软件，完全可用！

用户可以先用起来，后续再逐步添加AI能力
